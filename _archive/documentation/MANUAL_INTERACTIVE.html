📋 Übersicht

Was ist die Wallet GUI?

Die Wallet GUI ist eine grafische Benutzeroberfläche zum Scannen von Festplatten, Ordnern, Images und Geräten nach Kryptowährungs-Wallets und zugehörigen Daten.

Hauptfunktionen:

  • 🔍 Durchsuchen von Dateisystemen nach Wallet-Dateien
  • 💾 Unterstützung für verschiedene Formate (wallet.dat, Electrum, MetaMask, etc.)
  • 🔐 Erkennung von Mnemonics, Seeds und Private Keys
  • 📦 Auto-Mount für Images und Devices
  • 📊 Echtzeit-Monitoring und Ergebnisfilterung
  • 💼 Export-Funktionen für gefundene Treffer

🎯 Unterstützte Wallet-Typen

Wallet-Typ Erkennungsmuster Beschreibung
Bitcoin Core wallet.dat Standard Bitcoin-Wallet-Datei
Electrum default_wallet, *.electrum Electrum Wallet-Dateien
MetaMask metamask, vault MetaMask Browser-Extension
Mnemonics 12/24 Wörter BIP39 Seed-Phrasen
Keys xpub, xprv, WIF Erweiterte Public/Private Keys
Adressen bc1..., 0x... Bitcoin/Ethereum-Adressen

1

ROOT-Verzeichnis festlegen

Das ROOT-Verzeichnis ist Ihr Arbeitsordner, in dem alle Logs, Mounts und Staging-Bereiche erstellt werden.

  • Klicken Sie auf ROOT wählen…
  • Wählen Sie einen Ordner mit ausreichend Speicherplatz (empfohlen: externe Festplatte)
  • Standard: /run/media/emil/DATEN
2

Scan-Ziele hinzufügen

Legen Sie fest, welche Ordner, Dateien oder Geräte durchsucht werden sollen:

  • Ordner hinzufügen – Verzeichnis zum Durchsuchen
  • Datei/Abbild hinzufügen – Image-Dateien (.img, .dd, .vdi, etc.)
  • Device (/dev/…) hinzufügen – Physische Geräte/Partitionen

Tipp: Sie können mehrere Ziele gleichzeitig hinzufügen!

3

Scan-Optionen konfigurieren

Passen Sie das Scan-Verhalten an Ihre Bedürfnisse an (siehe Details unter "Optionen")

4

Scan starten

Klicken Sie auf Scan starten und beobachten Sie den Fortschritt im Live-Log.

5

Ergebnisse auswerten

Wechseln Sie zwischen den Tabs, um die gefundenen Wallets und Daten zu analysieren.

Zentrale Liste aller zu durchsuchenden Orte. Verwenden Sie die Buttons rechts, um Ziele zu verwalten:

1. Live-Log Tab

Zeigt die Echtzeit-Ausgabe des Scanners:

  • Aktuelle Aktion und Fortschritt
  • Gefundene Dateien während des Scans
  • Laufzeit-Timer (unten)
  • Status: "Bereit", "Läuft", "Fertig", "Abgebrochen"

Auto-Scroll: Das Log scrollt automatisch nach unten bei neuen Einträgen.

2. Hits Tab

Tabellarische Übersicht aller Treffer:

  • Spalten: Datei | Zeile | Treffer-Snippet
  • Filter: Regex-basiert (z.B. wallet\.dat|electrum|bc1)
  • Sortierung: Klick auf Spaltenköpfe zum Sortieren
  • Aktionen:
    • Öffnen – Öffnet Datei-Location in Dolphin
    • Export .tsv – Exportiert gefilterte Treffer als TSV
    • Kopieren – In Zwischenablage kopieren

3. Mnemonics Tab

Zeigt potenzielle Seed-Phrasen (12/24 Wörter):

  • Rohe Kandidaten aus dem Scan
  • Kann False Positives enthalten
  • Manuelle Überprüfung erforderlich

4. Anleitung Tab

Eingebettete Kurzanleitung innerhalb der GUI.

ROOT in Dolphin Öffnet das ROOT-Verzeichnis im Dateimanager
Staging öffnen Öffnet ROOT/Software/_staging_wallets/
Logs öffnen Öffnet das Logs-Verzeichnis oder den letzten Scan-Ordner
Scan starten Startet den Scan-Prozess
Stop Bricht den laufenden Scan ab (wird erst aktiv während Scan)

🔄 Detaillierter Arbeitsablauf

Szenario 1: Festplatte/Ordner durchsuchen

  1. Starten Sie die Wallet GUI
  2. Setzen Sie ROOT auf einen Ordner mit genug Speicherplatz
  3. Klicken Sie "Ordner hinzufügen" und wählen Sie das zu durchsuchende Verzeichnis
  4. Optional: Aktivieren Sie "Aggressiv" für gründlichere Suche
  5. Optional: Aktivieren Sie "Staging anlegen" für einfachen Zugriff
  6. Klicken Sie "Scan starten"
  7. Beobachten Sie den Fortschritt im Live-Log Tab
  8. Nach Abschluss: Wechseln Sie zum Hits Tab und filtern Sie die Ergebnisse
  9. Verwenden Sie "Öffnen" um gefundene Dateien zu inspizieren
  10. Exportieren Sie wichtige Funde mit "Export .tsv"

Szenario 2: Image-Datei (.img, .dd, .vdi) scannen

  1. ROOT festlegen
  2. "Datei/Abbild hinzufügen" klicken und Image auswählen
  3. Wichtig: "Auto-Mount" aktivieren (Standard)
  4. Falls Root-Rechte nötig: "Mit Root (pkexec)" aktivieren
  5. Scan starten – Image wird automatisch unter ROOT/_mount/ gemountet
  6. Nach dem Scan wird das Image automatisch unmountet

Szenario 3: Physisches Gerät/Partition scannen

  1. ROOT festlegen
  2. "Device (/dev/…) hinzufügen" klicken
  3. Gerät auswählen (z.B. /dev/sdb1)
  4. Erforderlich: "Mit Root (pkexec)" aktivieren
  5. Erforderlich: "Auto-Mount" aktivieren
  6. Scan starten – Sie werden nach Root-Passwort gefragt
  7. Gerät wird read-only gemountet und gescannt

⚠️ Wichtige Hinweise

  • Root-Rechte: Nur aktivieren, wenn nötig (Images/Devices)
  • Read-Only: Auto-Mount verwendet read-only Modus zur Sicherheit
  • Speicherplatz: Stellen Sie sicher, dass ROOT genug Platz für Logs hat
  • Laufzeit: Große Scans können Stunden dauern – nutzen Sie den Timer
  • Stop-Funktion: Sie können Scans jederzeit abbrechen

⚙️ Optionen im Detail

🔍 Aggressiv-Modus

Was macht es?

  • Erweitert Suchmuster um seltenere Wallet-Typen
  • Durchsucht mehr Dateitypen (Archive, Container, etc.)
  • Wendet tiefere Analyse-Methoden an
  • Sucht auch in versteckten/gelöschten Bereichen

Nachteile:

  • ⏱️ Deutlich längere Scan-Dauer
  • 💾 Mehr Ressourcenverbrauch (CPU/RAM)
  • 🔍 Mehr False Positives möglich

Wann verwenden? Bei wichtigen Wiederherstellungsversuchen oder wenn Standard-Scan nichts findet.

🔗 Staging anlegen

Funktion: Erstellt symbolische Links zu allen Funden in einem zentralen Ordner.

Location: ROOT/Software/_staging_wallets/

Vorteile:

  • 📁 Alle Funde an einem Ort
  • 🔗 Originaldateien bleiben unberührt
  • ✅ Einfaches Durcharbeiten der Treffer
  • 💼 Perfekt für weitere Verarbeitung/Backup

Struktur:

_staging_wallets/
├── bitcoin/
│   ├── wallet.dat -> /original/path/wallet.dat
│   └── ...
├── electrum/
│   └── default_wallet -> /original/path/default_wallet
└── metamask/
    └── vault -> /original/path/vault

🔐 Mit Root (pkexec)

Wofür? Ermöglicht Zugriff auf geschützte Bereiche und Geräte.

Erforderlich für:

  • Physische Geräte (/dev/sdX)
  • System-Partitionen
  • Geschützte Image-Dateien

⚠️ Sicherheitshinweis:

  • Sie werden nach Ihrem Root-Passwort gefragt
  • Scanner läuft mit erhöhten Rechten
  • Nur aktivieren, wenn wirklich nötig
  • Code ist Open Source und überprüfbar

💿 Auto-Mount

Automatisches Mounten von Images und Geräten

Unterstützte Formate:

  • Raw Images (.img, .dd, .raw)
  • Virtual Disks (.vdi, .vmdk, .vhd)
  • Physische Devices (/dev/sdX)
  • Archive (mit 7z/p7zip)

Mount-Optionen:

  • ro – Read-Only (keine Änderungen möglich)
  • noexec – Keine Ausführung von Programmen
  • nodev – Keine Device-Dateien

Location: ROOT/_mount/<name>/

Cleanup: Automatisches Unmount nach Scan-Ende

📊 Ergebnisse auswerten

🎯 Hits filtern (Regex-Beispiele)

Verwenden Sie den Filter im Hits-Tab, um spezifische Treffer zu finden:

Filter Findet
wallet\.dat Nur wallet.dat Dateien
electrum|wallet Electrum ODER wallet
bc1[a-z0-9]{39,} Bitcoin Bech32 Adressen
0x[a-fA-F0-9]{40} Ethereum Adressen
xpub|xprv Extended Keys
^(?!.*backup).*wallet Wallet, aber KEINE Backups

📤 Export-Funktionen

1. Export als TSV

  • Exportiert gefilterte Hits als Tab-Separated-Values
  • Öffnen mit: Excel, LibreOffice, Texteditor
  • Format: Datei | Zeile | Snippet
  • Verwendung: Dokumentation, Weitergabe, weitere Analyse

2. In Zwischenablage kopieren

  • Kopiert gefilterte Hits direkt in die Zwischenablage
  • Schnelles Einfügen in andere Tools
  • Format identisch zu TSV-Export

3. Einzelne Dateien öffnen

  • Wählen Sie einen Treffer in der Tabelle
  • Klicken Sie "Öffnen"
  • Dolphin öffnet sich mit markierter Datei
  • Direkter Zugriff auf Originaldatei

🎓 Erweiterte Funktionen

🔧 Scanner-Skript anpassen

Sie können eigene Scanner-Skripte verwenden:

  1. Erstellen Sie ein Bash-Skript mit der Signatur:
    script.sh ROOT [OPTIONS] TARGET [TARGET2 ...]
  2. Skript muss Ausgabe im Format schreiben:
    OUT=/pfad/zum/scan-ordner
  3. In GUI: "Scanner wählen…" und Ihr Skript auswählen

Beispiel-Struktur:

#!/bin/bash
ROOT="$1"
shift
# ... Optionen parsen ...
# ... Scan durchführen ...
echo "OUT=${OUTDIR}"  # Wichtig für GUI!

⌨️ Kommandozeilen-Aufruf

Scanner kann auch direkt über Terminal genutzt werden:

# Einfacher Scan
bash wallet_harvest_any.sh /mnt/daten /path/to/scan

# Mit Optionen
bash wallet_harvest_any.sh /mnt/daten --aggressive --staging /path/to/scan

# Mit Auto-Mount für Image
bash wallet_harvest_any.sh /mnt/daten --auto-mount image.dd

# Mit Root-Rechten
sudo bash wallet_harvest_any.sh /mnt/daten --auto-mount /dev/sdb1

🔧 Fehlerbehebung

Mögliche Ursachen & Lösungen:

  1. PyQt6 nicht installiert
    sudo apt install python3-pyqt6
  2. Python-Version zu alt
    Benötigt Python 3.8+ – prüfen mit python3 --version
  3. Fehler in der Konsole prüfen
    python3 ~/.local/share/wallet-gui/wallet_gui.py

Checkliste:

  • ✅ ROOT-Pfad existiert und ist beschreibbar?
  • ✅ Scanner-Skript existiert und ist ausführbar?
    chmod +x ~/.local/share/wallet-gui/scripts/wallet_harvest_any.sh
  • ✅ Bei Devices: "Mit Root" aktiviert?
  • ✅ Mindestens ein Ziel hinzugefügt oder Standard-Ziele vorhanden?

Mögliche Gründe:

  1. Scan noch nicht abgeschlossen
    Ergebnisse werden erst nach Scan-Ende geladen
  2. Keine Wallets gefunden
    Versuchen Sie "Aggressiv"-Modus
  3. Filter zu restriktiv
    Leeren Sie das Filter-Feld
  4. Logs prüfen
    Öffnen Sie ROOT/_logs/*/hits.txt manuell

Lösungsansätze:

  • Root-Rechte erforderlich:
    Aktivieren Sie "Mit Root (pkexec)"
  • Tools fehlen:
    sudo apt install sleuthkit ntfs-3g
  • Image beschädigt:
    Prüfen mit file image.dd
  • Bereits gemountet:
    sudo umount ROOT/_mount/*

Optimierungen:

  • Deaktivieren Sie "Aggressiv" für schnellere Scans
  • Grenzen Sie Ziele ein (nur relevante Ordner)
  • Schließen Sie Backup-Ordner aus
  • Nutzen Sie Filter statt alles zu scannen
  • Prüfen Sie Festplatten-Gesundheit (defekte Sektoren)

❓ Häufig gestellte Fragen (FAQ)

Ist die Wallet GUI sicher?

✅ Ja. Die GUI ist Open Source, liest nur Daten (außer Logs schreiben) und verwendet read-only Mounts für Geräte. Bei "Mit Root" sollten Sie dem Code vertrauen – er ist vollständig einsehbar.

Kann ich mehrere Scans parallel laufen lassen?

❌ Nein. Pro GUI-Instanz ist nur ein Scan gleichzeitig möglich. Sie können aber mehrere GUI-Fenster öffnen.

Werden Dateien verändert oder gelöscht?

❌ Nein. Der Scanner arbeitet read-only. Es werden nur Logs, Mounts und Staging-Links erstellt – niemals Original-Dateien modifiziert.

Was ist der Unterschied zwischen Hits und Mnemonics?

Hits: Alle Treffer mit Context (Datei, Zeile, Snippet) – z.B. wallet.dat Dateien, Adressen, Keys
Mnemonics: Spezielle Extraktion von Seed-Phrasen (12/24 Wörter) – nur Kandidaten, oft False Positives

Wie lange dauert ein typischer Scan?

⏱️ Abhängig von:

  • Datenmenge (1 TB = mehrere Stunden)
  • Aggressiv-Modus (2-5x langsamer)
  • Dateisystem (NTFS langsamer als ext4)
  • Festplatten-Geschwindigkeit (SSD >> HDD)

Faustregel: 100 GB normale Daten ≈ 15-30 Minuten (ohne Aggressiv)

Kann ich Scans pausieren und fortsetzen?

❌ Derzeit nicht. Sie können Scans stoppen, aber nicht pausieren. Ein Neustart beginnt von vorne.

Werden verschlüsselte Wallets erkannt?

⚠️ Teilweise. Verschlüsselte Wallet-Dateien (wie verschlüsselte wallet.dat) werden gefunden, aber der Inhalt kann nicht gelesen werden. Sie benötigen das Passwort zur späteren Entschlüsselung.

Wo finde ich die Scanner-Logs?

📁 ROOT/_logs/walletscan_<timestamp>/
Oder verwenden Sie den Button "Logs öffnen" in der GUI.

Unterstützt die GUI auch andere Betriebssysteme?

🐧 Linux: Vollständig unterstützt (getestet)
🪟 Windows: Theoretisch möglich mit WSL, aber nicht getestet
🍎 macOS: Theoretisch möglich, aber nicht getestet
